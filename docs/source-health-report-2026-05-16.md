# 数据源健康检查报告

检查时间：2026-05-16  
检查网络：当前开发机网络，时区 Asia/Hong_Kong  
检查关键词：`庆余年`  
检查范围：`SourceRegistry.kt` 中全部 32 个内置源

## 检查方法

每个源按以下链路检查：

1. API 搜索：`ac=list&wd=庆余年&pg=1&at=json`
2. 详情接口：使用搜索到的首个 `vod_id` 请求详情
3. 播放地址提取：从 `vod_play_url` 中提取首个直接媒体地址
4. 媒体可访问性：带浏览器 UA 和动态 Referer 请求首个 `.m3u8`/媒体 URL

对不支持关键词搜索但列表接口可用的源，使用首页列表首条记录作为降级样本，并在表格中标注。

## 对开发的影响

1. 当前“默认启用 5 源”质量不合适。建议把默认启用源改成稳定可播源组合，例如 `dbzy`, `dytt`, `jszy`, `hnzy`, `wjzy`, `rycj`。
2. `1080` 和 `yzzy` 的 API 问题是客户端可修的：需要为对应域名强制 HTTP/1.1。但它们的播放媒体仍然 403，修 API 后也不应作为默认播放源。
3. `ffzy`, `bdzy`, `ukzy` 这类媒体 403 是源/CDN/地区策略问题，不是简单请求头能解决。
4. `bfzy`, `lzzy` 是样本媒体 404，属于源数据陈旧或资源下线。
5. `myzy` 证书过期，不建议在客户端放开证书校验；可以作为“源异常”提示。
6. 搜索聚合需要区分“关键词搜索不可用”和“源不可用”。`yhzy` 列表和播放可用，但不适合当前关键词聚合逻辑。
